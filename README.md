# 📋 Research Synthesizer — PM Agent #2

A Streamlit app that synthesizes themes across user interview transcripts using Claude Sonnet 4.6.

Drop 3-5 transcripts. Get a structured synthesis report in under 30 seconds — themes ranked by signal strength, verbatim quotes pulled as evidence, cross-interview contradictions surfaced, open questions identified for follow-up research.

**Live demo:** [streammind-research-synthesizer.streamlit.app](https://streammind-pm2-research-synthesizer.streamlit.app/)

**Part of [StreamMind](https://github.com/shreya-patel-PM)** — a 22-week portfolio of 24 AI agents shipped across streaming, product management, and pharma domains, all without writing production code.

---

## What it does

A real user research synthesis pass — the kind a PM spends 4-6 hours doing manually — automated to under 30 seconds:

- **Cross-transcript theme detection** ranked by signal strength (a strong quote from 1 interview outranks vague mentions across 3)
- **Verbatim quote pulling** with attribution to the interviewee, not paraphrasing
- **Contradiction surfacing** across interviewees (essential for calibrating product decisions)
- **Open questions** the research raised but did not answer
- **Confidence calibration** based on sample size (refuses to overclaim with small samples)
- **Markdown export** for sharing with the team

---

## Why it matters

User research synthesis is one of the highest-value, lowest-leverage activities in PM work. The bottleneck isn't generating insights — it's the 4-6 hours of careful reading, coding, and pattern-matching that lets you write the synthesis memo nobody quite has time for.

This agent compresses that work without sacrificing the analytical rigor that makes it useful. The synthesis includes verbatim quotes (so the team can trace claims back to source), surfaces real contradictions (not just majority opinions), and refuses to overclaim with small samples.

## Running locally

Prerequisites: Python 3.10+ and an Anthropic API key from [console.anthropic.com](https://console.anthropic.com).

```bash
# Clone the repo
git clone https://github.com/YOUR_USERNAME/streammind-pm2-research-synthesizer.git
cd streammind-pm2-research-synthesizer

# Create a virtual environment
python -m venv venv
venv\Scripts\activate          # Windows
# source venv/bin/activate     # Mac/Linux

# Install dependencies
pip install -r requirements.txt

# Add your API key to .env
echo ANTHROPIC_API_KEY=sk-ant-api03-XXXX > .env

# Run the app
streamlit run app.py
```

The app opens at `http://localhost:8501`.

## Architecture

| Layer | Implementation |
|---|---|
| File parsing | python-docx (.docx), pypdf (.pdf), built-in (.txt) |
| Prompt construction | Structured system prompt with strict JSON schema |
| LLM call | Anthropic Python SDK · Claude Sonnet 4.6 · temperature 0.2 |
| Response parsing | Defensive parsing with markdown-fence stripping |
| UI | Streamlit with custom StreamMind theme |
| Export | Native Markdown rendering of synthesis JSON |

---

## Design decisions worth noting

**Temperature 0.2** — High enough for natural language variety in the synthesis prose, low enough for consistent JSON structure across runs.

**Defensive JSON parsing** — Claude reliably returns valid JSON ~90% of the time, but occasionally wraps responses in markdown fences (`` ```json ... ``` ``). The parser strips these before `json.loads`, so the agent works in the 10% failure case too.

**Verbatim quotes over paraphrase** — The schema instructs Claude to pull exact phrases from transcripts. This makes the synthesis traceable: every claim can be checked against source material. This is a real PM-quality difference from LLM-summary-style output.

**Confidence calibration** — With 3 transcripts, the agent correctly reports "Medium" confidence and warns against generalizing without a larger sample. Calibrated humility is more useful than overconfident takeaways.

**Cross-transcript synthesis, not per-transcript summaries** — The agent finds patterns ACROSS interviews. A theme appearing in 3 interviews has more weight than 3 unrelated themes each appearing in 1 interview. This is what makes it useful for PM work.
